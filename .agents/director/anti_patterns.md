# Bobrain Anti-Patterns

> 「これになったら却下」リスト。AI エージェントの機能追加・コピー生成・LP 改稿が、
> 平凡化のシグナルに該当した場合は QDAIF 評価以前に **即却下** する。

## カテゴリ 1: 思想を表に出してしまう（モード A 違反）

横断哲学 OS（memory `cross_project_philosophy`）の「思想は裏に置く」原則に反するパターン。

- ❌ LP hero に「あなたの脳を拡張する」「第二の脳」と書く（既出 + 説教臭い + Tiago Forte 二番煎じ）
- ❌ README の冒頭に「映す世界を間違えた」「解釈を変える」など哲学的フレーズを直接書く
- ❌ Quickstart の本文に作者の人生観・思想を混ぜる（footer の 1 行を除く）
- ❌ コミットメッセージに哲学を書く（事実報告に徹する）
- ✅ footer に "Made by ぼぶ — 1 行" を置く（人格を滲ませる正規ルート）

## カテゴリ 2: 平均値への収束（QDAIF 多様性低下）

学習データの平均値・既存競合の真似に陥るパターン。

- ❌ 「LangChain ベースの RAG」「OpenAI Embedding 使用」など平均的スタックへの降格
- ❌ 「ChatGPT のようにあなたのデータと対話」など、外部 LLM ラッパーの言葉づかい
- ❌ 「Notion AI / Obsidian Smart Connections / Mem.ai のような」と既存プロダクトと並べて訴求
- ❌ 自動要約・GPT 風応答機能の追加（chunk を返すだけの原則を破る）
- ❌ 「全てを記憶する」「無限スケール」「あらゆる」など最大化系のコピー（具体性ゼロ = 平均値）

## カテゴリ 3: ローカルファースト思想の侵食

bobrain 固有 philosophy_os の「自分の脳のデータを他人の API に預けない」原則に反するパターン。

- ❌ クラウド側でベクトル化する機能（OpenAI Embedding API 連携など）
- ❌ ユーザーの Vault 内容を匿名化した上でも統計収集する telemetry
- ❌ Pro 版機能として「クラウド同期」「マルチデバイス」を入れる（最後まで作らない）
- ❌ 「速度のためにクラウドオプションを」という妥協（速度問題は CoreML / 量子化で解く）
- ✅ Pro 版機能はあくまで「ローカルでより便利になる」方向のみ（チーム機能 / 高度な分析 / 業務統合）

## カテゴリ 4: 「便利ツール」化（変容的体験の喪失）

vision.md の「過去の自己との再会」を放棄して単なる便利ツールに堕すパターン。

- ❌ 「検索が高速」「軽量」「シンプル」だけを訴求（変容的体験ゼロ）
- ❌ ユーザーレビューで「便利」「悪くない」「使いやすい」しか出ない状態を放置
- ❌ 機能比較表の数値だけで競合に勝とうとする（速度・精度の数値競争に巻き込まれる）
- ✅ 「過去の自分が、今の自分より良いことを言っていた」という体験談を意図的に集める

## カテゴリ 5: ぼぶ人格の侵襲過剰（モード A 越境）

逆方向の失敗。プロダクト発信物に人格を入れすぎるパターン。

- ❌ README の本文に「俺は」「あんた」を入れる（footer の 1 行のみ）
- ❌ Issue 返信で B モード（フル人格）を使う（A モード = 中立で対応）
- ❌ 赤ちゃんアバター動画オーバーレイを LP に入れる（外部レポートの不採用例、既出）
- ❌ 思想的 README リライト（外部レポートの不採用例、既出）
- ✅ Show HN / Reddit ローンチ投稿は「機能 → 1 行人格」の構成（最初は中立）

## カテゴリ 6: アプリツリー固有のアンチパターン

横断ルールへの違反。

- ❌ Bob 以外のキャラを Bobrain の作品キャラとして追加する（モチーフキャラ原則: 1 アプリ 1 キャラ）
- ❌ Bob を「ぼぶ作者人格」と混同したコピーを書く（OSS Linus と Tux の関係を維持）
- ❌ bob-universe 系の「沈黙モード」表現を bobrain に流用する（character-universe / bob-universe の境界を尊重）

## 検出時のフロー

1. 機能追加・コピー変更案を評価
2. 上記 6 カテゴリのいずれかに該当 → **即却下、AgentCouncil に差し戻さない**（Gamma の判定以前）
3. カテゴリに該当しない → QDAIF 評価軸（`qdaif_axes.yaml`）でスコアリング
4. スコア閾値 14 未満 → 却下 / 14-18 → AgentCouncil 議論 / 18 以上 → 採用候補
