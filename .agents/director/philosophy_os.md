# Bobrain Philosophy OS

Bobrain のあらゆる設計判断・コピー・機能追加の評価関数として AI エージェント（Claude Code / subagents）の基底に注入する哲学 OS。

## ベースは横断 OS への参照

bobrain 固有の哲学を新規定義しない。横断 OS を継承する：

- **memory `cross_project_philosophy.md`** — 「映す世界を間違えた」/ 解釈を変える / 裏テーマ世界平和。**全プロジェクト共通で裏に置く**（philosophy-chat 本体のみ表に出してよい）
- **memory `bob_persona.md`** — 作者ぼぶ人格（モード A/B 分離、サングラス赤ちゃんアバター、ぼぶ表記）
- **アプリツリー wiki [[映す世界を間違えた]]** — 横断哲学の正本
- **`~/projects/CLAUDE.md`** — ハブ（自動 context 投入）

## bobrain 固有の上書き

横断 OS に対し、bobrain でだけ強調する観点：

### 1. 「忘却 = 消失」ではなく「忘却 = 再発見の余地」

- Bobrain の設計は、過去の自己との再会を扱う
- 外部 LLM が「答えを生成する装置」だとすれば、Bobrain は「自分が既に持っている答えを掘り返す装置」
- Vault と repo を横断するのは、知の集合に対する検索ではなく、**過去の自己の足跡** に対する検索

### 2. ローカルファースト = 思想

- 「速度のためのローカル」ではなく「**自分の脳の延長を他人の API に預けない**」という姿勢
- e5-large の CPU 推論が遅くても、それを許容することで「自分の脳のデータが自分の手元から動かない」という絶対の前提を担保する
- これはマネタイズ判断（Pro 版機能の選定）の最終フィルタにも使う：**Pro 版でも「データを外に出す機能」は最後まで作らない**

### 3. 検索結果は「答え」ではなく「手がかり」

- Bobrain は LLM ではない。chunk を返すだけで要約しない（要約は呼び出し側 LLM の責務）
- これは「ユーザーが自分の言葉で過去の自分と対話する余地」を残すため
- 自動要約 / GPT 風応答化は **anti_patterns.md** の対象

## 矛盾チェック

bobrain の機能追加・コピー変更時、上記 3 つに矛盾しないか自問：

- [ ] この機能は「外部 LLM の代替」ではなく「過去の自己との再会」を支援しているか
- [ ] この機能はデータをローカルに留めているか
- [ ] この機能はユーザーが自分の言葉で考える余地を奪っていないか

矛盾する場合、機能を削るか、思想を見直す。両方できないなら **作らない**（横断 OS の「迷ったら書かない」原則）。

## モード A/B との関係

- bobrain README / LP / Quickstart はモード A（中立、機能ドリブン）。本ファイルの内容を **直接書かない**
- 表に出てよいのは footer の "by ぼぶ" + 1 行のみ
- Show HN / Reddit ローンチ投稿で本ファイルの内容を語る場合もモード A 寄り（最初の 1 段は機能、最後に 1 行で人格）
- philosophy-chat 本体だけが例外（Bobrain は例外ではない）
