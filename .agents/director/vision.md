# Bobrain Vision

> Bobrain が提供すべき変容的体験（Transformative Experience）— ユーザーが Bobrain を使った後、世界の見え方がどう変わるべきか。

## 核（1 段落）

Bobrain は「自分の脳の検索インデックス」ではない。**「自分が過去に何を見て、何を考え、何を言葉にしてきたか」を再発見する装置**。ユーザーは Bobrain を使うことで、忘れていた自分の過去の思考に出会い、「自分が今考えていることは、過去の自分が既にもっと深く考えていた」という不可逆な気付きを得る。これは外部知識のクエリではなく、**自己と過去の自己との対話**。

## ユーザーが Bobrain 使用前に持っている前提

- 「過去のメモは検索できないから死んでいる」
- 「自分の知識は今手元にあるものが全て」
- 「外部 LLM の方が自分のメモより賢い」

## ユーザーが Bobrain 使用後に到達すべき認識

- 「過去の自分は、今の自分より良いことを言っていることがある」
- 「自分のメモと外部 LLM は同じクエリで横並びに参照できる」
- 「忘れることは、消失ではなく、再発見の余地である」

## このプロダクトが避けるべき方向

- 単なる「Obsidian の高速検索」として位置付けること（=機能の説明にしか聞こえない）
- 「AI が要約してくれる便利ツール」として売ること（= chunked summarization の量産品に埋もれる）
- 「あなたの第二の脳」と謳うこと（= Tiago Forte 系の二番煎じ、既に飽和）

## 表現の鉄則（モード A 準拠）

LP / README / narration では、上記の核を **直接書かない**。機能説明（Obsidian と repo の横断検索 / hybrid search / local-first）に徹し、変容的体験は **footer の "by ぼぶ" + 1 行** にだけ滲ませる。書き手の駆動力として使う。

## 変容の証拠（プレイテスト的検証ポイント）

ユーザーが Bobrain を 1 週間使った後、次のいずれかを自発的に言ったら成功：

1. 「過去の自分が今の悩みに既に答えていた」
2. 「Vault と repo を同じクエリで叩けるのは思ってたより便利」
3. 「LLM に聞く前に Bobrain を叩く習慣ができた」

逆に「なんとなく便利」「悪くない」しか出なかったら、体験が平凡化している = L4 Playable Gate で差し戻し対象。
